El Problema: Antes de GitHub

😱

versión_final_2.txt

¿Te suena familiar tener 10 versiones de un archivo con nombres como "final", "final_final", "final_definitivo"?

💥

Sobrescribir cambios

Trabajar en equipo era un caos. ¿Quién tiene la última versión? ¿Se perdieron mis cambios?

🔍

Sin historial

¿Cuándo se rompió el código? ¿Quién hizo ese cambio? Imposible saberlo.

Necesitamos una forma de guardar "fotos" de nuestro código en el tiempo...

Git vs GitHub

No son lo mismo, pero trabajan juntos

Git

Git

Software local

  • Se instala en tu computadora
  • Control de versiones local
  • Línea de comandos (CLI)
  • Creado por Linus Torvalds (creador de Linux)

GitHub

Plataforma web

  • Servicio en la nube (internet)
  • Interfaz visual amigable
  • Colaboración en equipo
  • Comunidad global (Microsoft)

Git es el motor, GitHub es el garaje donde guardas y muestras tu auto

Conceptos Fundamentales

📁

Repositorio

La carpeta de tu proyecto

Es como una carpeta mágica que guarda todo el historial de cambios, no solo los archivos actuales.
💾

Commit

Foto del código

Un "punto de guardado" con mensaje descriptivo. Ej: "Agregué función de login".
🌿

Branch

Línea temporal paralela

Rama alternativa para experimentar sin romper el código principal (main).
🔀

Merge

Fusionar cambios

Unir tu rama experimental de vuelta a la rama principal cuando todo funciona.
terminal

➜ mi-proyecto git:(main) git add .

➜ mi-proyecto git:(main) git commit -m "Mi primer commit"

➜ mi-proyecto git:(main) git push origin main

El Flujo de Trabajo

1. Editar

Modificas archivos en tu editor

→

2. Stage

git add (preparar)

→

3. Commit

git commit (fotografiar)

→

4. Push

git push (subir)

Analogía: Fotografía

Stage (add): Como preparar a tus amigos para la foto.
Commit: Como tomar la foto con un título.
Push: Como subirla a Instagram para que todos la vean.

Analogía: Videojuegos

Commit: Es un punto de guardado.
Push: Subir tu partida guardada a la nube.
Pull: Descargar la partida de un amigo para jugar juntos.

Creando tu Primer Repo

📚

mi-primer-proyecto

Público • Creado hoy

⭐ Star 🔀 Fork

# Inicializar repositorio local

git init


# Conectar con GitHub (remoto)

git remote add origin https://github.com/tu-usuario/mi-primer-proyecto.git


# Subir cambios

git add .

git commit -m "Primer commit"

git push -u origin main
